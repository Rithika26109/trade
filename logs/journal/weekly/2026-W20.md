## What worked

- **Monday's no-trade discipline.** MIN_CONFIRMATIONS=2 enforced in code for
  the first full session — 8 signals rejected without a single bypass leak.
  TREND_DOWN regime call was accurate and the right response was to sit out.
  Capital preserved on a pre-CPI day.
- **Tuesday's three-for-three.** The week's only profitable session was also
  the only one where every entry cleared the 2+ confirmation gate honestly.
  Small sample but the pattern fits: process-clean entries make money.
- **Hard-avoid blocks held when respected.** Fri SAIL (Q4 board meet), BPCL
  (trading window closed), TATAPOWER's wide-ORB exclusion on Wed (-6.55% gap)
  all worked as designed. The plan-side blocks are reliable when the bot
  honors the watchlist.
- **Default `both` bias.** Last week's W19 promoted heuristic was applied
  every day this week. Zero bias-veto losses-of-opportunity — versus W19's
  13 (Wed) + 5 (Fri). The directional-bias problem is solved at the prompt
  level.

## What didn't

- **HIGH_CONVICTION_SCORE bypass is now the biggest single cost.** 1-strategy
  entries fired on Wed (×4), Thu (×5), Fri (×3 of 4) — 12 of 14 entries on
  the three losing sessions. Every #lesson tag since May 11 says the same
  thing. The W19 summary flagged it; it has now produced ~Rs 1,750 of paper
  losses in W19+W20 combined. Top-priority code fix.
- **`max_trades` override silently ignored.** Thu's plan said
  `max_trades=3` (carried over from Wed); bot took 5. Per current premarket.md
  step 3, this field is not honored — fine, but the plan should stop setting
  it and rely on `MAX_TRADES_PER_DAY` plus tighter `max_open_positions` /
  `risk_per_trade_pct`. The empty override is misleading post-mortem.
- **Stale daily_plan.** Thu ran on Wed's plan (no May 14 plan was generated).
  Premarket routine must produce a fresh file every trading day, and the
  bot/healthcheck should refuse to launch if `daily_plan.date != today`.
- **SELL-into-recovery losses.** Wed (3 SELL ORB entries into NIFTY +0.44%)
  and Thu (3 SELL entries into NIFTY +1.02%) — all six stopped out. ORB
  breakdown signals on a rising tape are systematically low-quality.
- **Post-crash entries had no ORB.** Fri 09:40 DNS crash, 09:45 restart,
  09:46 entries on HDFCBANK and HCLTECH (1-strategy each). Bot needs a
  post-restart entry cooldown. Code fix, not prompt.
- **Partial-qty EOD square-off bug still firing.** Fri TATAPOWER 74→56
  closed, 18 orphaned. Third consecutive week.

## Changes to premarket.md

Four edits applied this week. All are additive, justified by 2+ sessions of
W19+W20 evidence, and respect the HARD RULES block.

1. **Step 2 (research) — Gemini sector-cluster question.** Added an explicit
   prompt to ask "which other companies in the same sector print Q4 results
   today?" This was a W19 proposal; W20 confirmed the need when Thu's SBIN
   AGM contaminated the banking watchlist (HDFCBANK, RELIANCE side-trades).
2. **Step 3 (decision) — bias default and stale-bias check.** Made the
   `both`-by-default heuristic explicit (W19 proposal applied), and added a
   one-liner that any symbol carried over from yesterday must have today's
   tape justify its bias.
3. **Step 3 (decision) — banking-cluster rule.** When SBIN/BANKBARODA/KOTAK/
   ICICI/AXIS/HDFC print Q4 on the same day, keep at most one bank in the
   watchlist. W19 + W20 both demonstrated cluster volatility leaking.
4. **Step 3 (decision) — counter-tape SELL ORB caveat.** New W20 evidence:
   if NIFTY is recovering >+0.3% by 10:00, SELL ORB breakdown signals are
   low-quality and the plan should call this out in `notes` so the strategist
   does not over-weight `short`-leaning catalysts at open.

Also tightened the "open code bugs" guidance: every plan's rationale must
now list the HIGH_CONVICTION_SCORE bypass and `max_trades` non-enforcement as
known open bugs so the user sees them every morning until fixed.

**Not changed:** HARD RULES block (verbatim preserved), step counts, file
paths, Telegram step. No stylistic rewrites.

same and the prompt does not need further tinkering — the bypass needs
